# Change Protocol

Before non-trivial changes, explain: **What** (files), **Why** (rationale), **Alternatives** (options table), **Effects** (breaking changes, tests).

Skip for: typo fixes, comments, formatting, or when user says "just do it".

## TSG Format (Troubleshooting)

```markdown
## Issue
[One-line problem]

## Root Cause
[1-2 sentences]

## Fix
[Action taken]

## Verify
[Command to confirm]
```

## Post-Change

Summarize: what was done, how to verify, any follow-up needed.
